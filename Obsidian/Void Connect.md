**Void Connect** - программа для создания децентрализованной сети, основанной на локальном подключении компьютеров. По сути — мини-интернет внутри локальной сети, вдохновлённый архитектурой BitTorrent.

## Цель проекта

Объединить устройства в локальную сеть без центрального сервера. Пользователи смогут общаться, обмениваться файлами, публиковать сайты и настраивать профили. Каждый участник вносит вклад в поддержание сети — как в торренте.

## Основные компоненты

### Сеть и безопасность

- [[Обнаружение узлов]] — автоматический поиск участников через mDNS
- [[Глобальная сеть]] — выход за пределы LAN через bootstrap-узлы
- [[Безопасность сети]] — защита от атак изнутри сети, шифрование
- [[Система репутации]] — стимулирование вклада, изоляция нарушителей

### Коммуникация

- [[Общий чат]] — чат для всех участников сети в реальном времени
- [[Личный чат]] — приватное общение с E2E шифрованием

### Данные и хостинг

- [[Общее хранилище данных]] — P2P обмен файлами по схеме чанков
- [[Поднятие простых сайтов]] — хостинг статических сайтов с распределённым кэшированием
- [[Внутренний DNS]] — адреса вида `vasya.void` вместо IP

### Профиль

- [[Кастомизация аккаунта]] — имя, аватар, статус; публичный ключ как ID

## Порядок разработки

1. **[[Обнаружение узлов]]** — mDNS, список пиров 1.5. **[[Глобальная сеть]]** — bootstrap-узлы, NAT traversal
2. **[[Общий чат]]** — базовый, без шифрования
3. **[[Кастомизация аккаунта]]** — ключи, профили (фундамент для всего)
4. **[[Личный чат]]** — P2P + E2E шифрование
5. **[[Безопасность сети]]** + **[[Система репутации]]**
6. **[[Общее хранилище данных]]** — чанки, SHA-256, раздача
7. **[[Поднятие простых сайтов]]** — HTTP-сервер, распределённый хостинг
8. **[[Внутренний DNS]]** — зона .void

## Рекомендуемый стек

- **Язык**: Rust
- **Async runtime**: Tokio
- **Сетевой уровень**: TCP, WebSocket (`tokio-tungstenite`)
- **Обнаружение устройств**: mDNS (`mdns-sd`) + UDP broadcast как резерв
- **Шифрование**: libsodium (`sodiumoxide`)
- **Веб-сервер**: Axum
- **База данных**: SQLite (`sqlx`)
- **Передача данных**: JSON или MessagePack
