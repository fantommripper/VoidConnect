Личный чат поддерживает приватное общение между двумя или несколькими выбранными участниками.

## Как можно реализовать

- Использовать **отдельные прямые каналы** между узлами (P2P, без ретранслятора)
- Для передачи сообщений: TCP, QUIC или WebSocket
- Обязательное **end-to-end шифрование**

## Схема шифрования

1. При первом запуске каждый пользователь генерирует пару ключей (публичный + приватный) — через `libsodium` или `ring`
2. Публичные ключи распространяются по сети как часть [[Кастомизация аккаунта|профиля]]
3. Сообщение шифруется **публичным ключом получателя**
4. Расшифровать может только получатель своим приватным ключом

Это та же схема, что использует Signal и большинство современных мессенджеров.

## Идентификация пользователей

Пользователи идентифицируются по своему публичному ключу — он же служит уникальным ID в сети.
