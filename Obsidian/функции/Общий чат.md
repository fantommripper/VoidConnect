Общий чат позволяет всем участникам сети отправлять и получать сообщения в реальном времени.

## Как можно реализовать

- Использовать **WebSocket поверх TCP** для постоянного соединения между клиентами
- Для обнаружения участников использовать [[Обнаружение узлов]] (mDNS)
- Сообщения передаются через схему с **временным ретранслятором**

## Схема ретранслятора

Один из узлов берёт на себя роль хаба и пересылает сообщения всем остальным. Если хаб падает — сеть автоматически выбирает нового (например, узел с наибольшим uptime или первый по алфавиту ID).

Это похоже на то, как работают торрент-трекеры — один координирует, но сеть не умирает без него.

## Альтернатива: P2P flood

Каждый узел получает сообщение и пересылает его всем, кого знает. Требует sequence numbers, чтобы одно сообщение не гуляло по кругу. Сложнее, но нет единой точки отказа.

## Надёжность

- Подтверждение доставки сообщений
- Повторная отправка при потере пакетов
- Sequence numbers для дедупликации

## Каналы («доски»)

Поверх единого общего чата работают **каналы** (по аналогии с досками форчана):
глобальный канал остаётся, плюс тематические. Это **не** отдельные сети —
технически канал это просто **тег** у сообщения.

### Как устроено

- У каждого сообщения есть поле `channel` (строка-идентификатор, напр. `tech`).
- Сообщения по-прежнему **рассылаются всем** через тот же ретранслятор —
  каналы не меняют сетевую схему. Фильтрация по каналу происходит **в интерфейсе**:
  показываются только сообщения выбранной доски.
- Поле `channel` **входит в подпись** сообщения вместе с отправителем, временем,
  seq и текстом — поэтому ретранслятор не может «переложить» чужое сообщение в
  другой канал, не сломав подпись.
- Канал сохраняется в БД (`public_messages.channel`, миграция `0002`), поэтому
  история раскладывается по доскам и переживает перезапуск. Пакеты/строки без
  поля канала считаются глобальными (`global`) — обратная совместимость.

### Каналы по умолчанию

Стартовый набор (id → название):

`global` Глобальный · `tech` Технологии · `dev` Разработка · `games` Игры ·
`music` Музыка · `movies` Кино · `art` Творчество · `help` Помощь ·
`random` Random · `trade` Барахолка.

Список задаётся в интерфейсе (`ui/src/pages/chat.rs`, константа `CHANNELS`) —
добавить доску = одна строка. Идентификатор канала хранится в сообщении, а
иконка и название — только для отображения.

### Ограничения и развитие

- История загружается единым окном последних N сообщений по всем каналам, потом
  раскладывается по доскам в UI. Очень активная доска может вытеснять из окна
  старые сообщения тихих досок (на больших объёмах стоит грузить историю
  отдельно по каналу).
- Возможные следующие шаги: значок непрочитанного на доске, создание
  пользовательских каналов, модерация на уровне канала.
