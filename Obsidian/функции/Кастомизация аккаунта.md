Пользователь настраивает свой профиль внутри локальной сети. Профиль служит одновременно идентификатором и визитной карточкой.

## Как можно реализовать

- Хранить профиль пользователя в **локальной базе данных** (SQLite, RocksDB или файловое хранилище)
- Поддержать изменение: имени, аватара, статуса, цвета темы, описания профиля
- Синхронизацию профилей между устройствами делать через локальный обмен метаданными
- Для авторизации использовать ключи или токены

## Структура профиля

Профиль — это подписанный JSON:

```json
{
  "id": "публичный_ключ",
  "name": "Vasya",
  "avatar": "base64...",
  "status": "online",
  "description": "Просто человек",
  "dns_name": "vasya.void",
  "updated_at": 1234567890,
  "signature": "подпись_приватным_ключом"
}
```

Подпись приватным ключом гарантирует, что никто не может подделать чужой профиль.

## Ключи как идентификатор

- При первом запуске генерируется пара ключей (публичный + приватный)
- **Публичный ключ** = уникальный ID пользователя в сети
- **Приватный ключ** хранится только локально, никогда не покидает устройство
- Публичный ключ распространяется по сети и используется для [[Личный чат|шифрования личных сообщений]]

## Связанные компоненты

- [[Личный чат]] — публичный ключ из профиля используется для шифрования
- [[Внутренний DNS]] — `dns_name` из профиля регистрируется как DNS-адрес
- [[Обнаружение узлов]] — профиль распространяется вместе с объявлением узла
