Участники сети могут размещать статические сайты (HTML + CSS + JS) внутри локальной сети. Сайты остаются доступны, даже если их владелец отключился.

## Как можно реализовать

- Каждый пользователь может запускать **локальный HTTP-сервер** на своём устройстве
- Для этого подойдёт встроенный веб-сервер на Rust: **Axum**, Actix Web или Hyper
- Сайты публикуются через внутренний адрес, имя узла или [[Внутренний DNS
- Для динамического контента можно поддержать шаблонизаторы и API через REST или WebSocket

## Внутренний DNS

Цель: вместо `192.168.1.5:8080` в браузере писать `vasya.void`.

- Каждый узел регистрирует своё имя в распределённой таблице имён
- Таблица синхронизируется между участниками через [[Обнаружение узлов]]
- Резолвинг происходит локально на устройстве пользователя

## Распределённый хостинг

**Проблема**: если владелец сайта отключается — сайт умирает.

**Решение**: другие узлы кэшируют файлы сайта и продолжают его раздавать.

### Как это работает

1. Файлы сайта (HTML/CSS/JS) хранятся как чанки — так же как в [[Общее хранилище данных]]
2. При публикации сайта владелец объявляет: "мой сайт состоит из файлов с хэшами X, Y, Z"
3. Желающие узлы подписываются на хостинг этого сайта и скачивают его файлы
4. Когда владелец уходит офлайн — подписанные узлы продолжают отдавать файлы посетителям
5. Посетитель получает файлы с любого доступного узла-хостера

### Каталог сайтов

- В сети ведётся общий каталог доступных сайтов
- В каталоге указано: название, владелец, список активных хостеров, статус (онлайн/офлайн)
- Посетитель всегда подключается к ближайшему активному хостеру

### Ограничения

- Работает только для **статических сайтов** (HTML + CSS + JS)
- Динамический контент (базы данных, серверная логика) так распределить нельзя — для него владелец должен быть онлайн

## Связанные компоненты

- [[Общее хранилище данных]] — файлы сайта хранятся по той же схеме чанков
- [[Обнаружение узлов]] — для публикации адреса сайта в сети
- [[Внутренний DNS]] — для удобных адресов вида `vasya.void`
