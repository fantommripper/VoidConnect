# Void Connect

**Децентрализованная P2P-сеть — мини-интернет внутри локальной сети.**
Вдохновлён архитектурой BitTorrent: нет центрального сервера, каждый участник
вносит вклад в работу сети. Внутри — общий и личный чат, обмен файлами, хостинг
сайтов, внутренний DNS (`.void`) и система репутации.

> Версия: **0.1.0** · Язык: **Rust** · Интерфейс: **egui/eframe** · Платформы: **Linux, Windows**

## Что это такое

Void Connect объединяет несколько компьютеров в **общую сеть без центрального
сервера**. Достаточно, чтобы машины «видели» друг друга по сети — в одной Wi-Fi,
через кабель, или через виртуальную сеть (Hamachi / Radmin VPN / интернет).

Что умеет:

- 💬 **Общий чат** — для всех участников сети, с **каналами** («досками» в духе форчана: Технологии, Игры, Музыка и т.д.); сообщения подписаны ключом отправителя.
- 🔒 **Личные сообщения** — приватный P2P-чат со сквозным (E2E) шифрованием.
- 🗄 **Хранилище** — обмен файлами по схеме чанков (как торрент): скачал → раздаёшь.
- 🌐 **Сайты** — публикация статических сайтов, доступных другим участникам.
- 🧭 **Внутренний DNS** — адреса вида `vasya.void` вместо `192.168.1.5:7704`.
- ⭐ **Репутация** — поощряет вклад в сеть и автоматически изолирует нарушителей.
- 🧑 **Профиль** — имя, аватар, статус, описание; публичный ключ как ID.

Данные всегда живут **на устройствах пользователей**, а не на сервере. Даже
глобальный режим (через интернет) использует «точки входа» только для первого
знакомства узлов — дальше трафик идёт напрямую.

## Интерфейс

Слева — боковое меню со страницами:

| Страница | Что внутри |
|---|---|
| **Общий чат** | Лента сообщений всей сети. Клик по аватару — открыть личный чат или пожаловаться. |
| **Личные сообщения** | Список диалогов и приватная переписка с E2E-шифрованием. |
| **Хранилище** | Опубликовать файл, скачать чужой, видеть прогресс и число сидеров. |
| **Сайты** | Опубликовать папку со статическим сайтом, открыть чужие сайты в браузере. |
| **Граф сети** | Визуальная карта узлов сети вокруг вас. |
| **Профиль** | Имя/аватар/статус, ваш `.void`-адрес, репутация, статистика, **ручное подключение** и проверка портов. |

Внизу — строка статуса: режим сети (Local / LAN), число узлов, ваш адрес и
предупреждение, если входящие порты закрыты.

## Установка и запуск

Проект собирается через **Nix** (всё окружение описано в [`shell.nix`](shell.nix)),
но подойдёт и обычный toolchain Rust + системные библиотеки.

### Вариант А. Из исходников (разработка)

```bash
# 1. Войти в окружение со всеми зависимостями (Rust, X11/Wayland, OpenGL, GTK)
nix-shell

# 2. Запустить графическое приложение
cargo run -p void-ui

# С логами обнаружения узлов:
RUST_LOG=info cargo run -p void-ui

# С именем и портом:
cargo run -p void-ui -- Vasya 7700
```

> Без Nix понадобятся: Rust (stable), а также `libX11/libxcb`, `wayland`,
> `libGL`, `gtk3`, `pkg-config` — полный список см. в [`shell.nix`](shell.nix).

### Вариант Б. Готовый бинарник

В репозитории есть [`Makefile`](Makefile) для сборки переносимых артефактов:

```bash
nix-shell
make linux      # нативный бинарник для Linux  → dist/
make appimage   # AppImage (один файл, запускается «как есть»)
make windows    # .exe для Windows (нужен cargo-zigbuild или mingw-w64)
make all        # всё сразу
```

Готовые файлы появятся в папке `dist/`. Бинарник называется **`void-connect`**.

## Быстрый старт: одна Wi-Fi / один роутер

Самый простой случай — все компьютеры в одной локальной сети (один роутер/точка
доступа).

1. Запусти Void Connect на каждом компьютере:
   ```bash
   cargo run -p void-ui -- Имя
   ```
2. Узлы **находят друг друга автоматически** (mDNS + UDP-broadcast). Через
   несколько секунд в строке статуса появится число узлов, а на странице
   **Граф сети** — соседи.
3. Пиши в **Общий чат**, открывай **Личные сообщения**, публикуй файлы и сайты.

Если узлы не появились — частые причины и решение в разделе
[Подключение вручную](#подключение-вручную-если-узлы-не-нашлись-сами) и
[Порты и файрвол](#порты-и-файрвол).

> ⚠️ Если на роутере включена **изоляция клиентов** (AP/Client Isolation) —
> устройства не видят друг друга на уровне железа. Автопоиск не сработает;
> используйте VPN-сеть (ниже) или ручное подключение.

## 🥿 Гайд для тапочек: своя сеть через Hamachi

Hamachi создаёт **виртуальную локальную сеть поверх интернета** — для Void
Connect всё выглядит так, будто вы в одной Wi-Fi, хотя физически вы в разных
городах. Бесплатно — до 5 компьютеров в сети.

### Шаг 1. Установить Hamachi на ВСЕХ компьютерах

1. Скачайте LogMeIn Hamachi с официального сайта **vpn.net**.
2. Установите и запустите. Зарегистрируйте/войдите в аккаунт LogMeIn.
3. Нажмите кнопку **включения** (▶) — Hamachi выдаст вашему ПК виртуальный
   IP-адрес вида **`25.x.x.x`**.

### Шаг 2. Создать сеть (делает один человек — «хост»)

1. Меню **Сеть → Создать новую сеть…**
2. Придумайте **идентификатор сети** и **пароль**.
3. Сообщите идентификатор и пароль друзьям.

### Шаг 3. Подключиться к сети (остальные)

1. Меню **Сеть → Подключиться к существующей сети…**
2. Введите идентификатор и пароль от хоста.
3. В окне Hamachi появятся все участники с их адресами `25.x.x.x` (зелёный
   кружок = на связи).

### Шаг 4. Запустить Void Connect

Запускайте **обычно**, без флага `--local`:

```bash
cargo run -p void-ui -- Имя
```

- Если автопоиск сработал — узлы появятся сами. Готово.
- Если за 15–20 секунд никого нет (Hamachi не всегда пропускает broadcast) —
  подключитесь вручную: см. следующий раздел. Адрес друга — это его адрес
  **`25.x.x.x`** из окна Hamachi плюс порт `:7700`, например `25.13.40.7:7700`.

> 💡 На странице **Профиль** в поле «Мой адрес» показывается ваш *физический*
> LAN-IP, а не Hamachi-адрес. Чтобы вас нашли через Hamachi, диктуйте друзьям
> именно адрес **`25.x.x.x`**, который виден в окне Hamachi.

> 🔥 Windows при первом запуске спросит про файрвол — нажмите **«Разрешить
> доступ»** (для частных и публичных сетей). Подробнее — [Порты и файрвол](#порты-и-файрвол).

## 🥿 Гайд для тапочек: своя сеть через Radmin VPN

Radmin VPN — бесплатная альтернатива Hamachi **без ограничения на число
участников**. Часто удобнее, потому что лучше пропускает широковещательный
трафик (то есть автопоиск узлов в Void Connect нередко работает сразу).

### Шаг 1. Установить Radmin VPN на ВСЕХ компьютерах

1. Скачайте Radmin VPN с официального сайта **radmin-vpn.com** (полностью бесплатно).
2. Установите и запустите. Регистрация не требуется.
3. Программа сама подключится; вашему ПК назначится адрес вида **`26.x.x.x`**.

### Шаг 2. Создать сеть (один человек)

1. Кнопка **Сеть → Создать сеть**.
2. Введите **имя сети** и **пароль**.
3. Передайте имя и пароль друзьям.

### Шаг 3. Подключиться (остальные)

1. Кнопка **Сеть → Присоединиться к сети**.
2. Введите имя и пароль.
3. В окне появятся все участники с адресами `26.x.x.x`.

### Шаг 4. Запустить Void Connect

```bash
cargo run -p void-ui -- Имя
```

- Чаще всего узлы находятся **автоматически** — Radmin VPN пропускает broadcast.
- Если нет — подключитесь вручную к адресу друга `26.x.x.x:7700` (см. ниже).

> 🔥 Не забудьте разрешить `void-connect` в файрволе на каждом ПК.

## Подключение вручную (если узлы не нашлись сами)

Автообнаружение может не сработать: изоляция клиентов на роутере, разные
подсети, VPN без broadcast. На этот случай есть ручное подключение.

1. Узнайте у друга его **IP-адрес** и порт:
   - в одной локалке — его LAN-IP (он виден на его странице **Профиль** →
     «Мой адрес»), например `192.168.1.42:7700`;
   - через Hamachi/Radmin — его VPN-адрес `25.x.x.x` / `26.x.x.x` плюс `:7700`.
2. Откройте страницу **Профиль → «Подключение к пиру вручную»**.
3. Введите адрес в формате `IP:порт` (например `26.8.15.3:7700`) и нажмите
   **«Подключиться»**.
4. Кнопка **«Тест TCP»** проверит, открыт ли у друга порт чата (если соединение
   не идёт — почти всегда виноват файрвол, см. ниже).

Достаточно, чтобы **хотя бы один** участник вручную подключился к другому —
дальше узлы обменяются списком пиров и сеть «соберётся» сама.

## Глобальная сеть (через интернет)

Помимо VPN, у Void Connect есть **родной механизм выхода в интернет** через
*bootstrap-узлы* — публичные «точки входа». Это тот же подход, что у Bitcoin и
торрентов: сервер нужен только для первого знакомства, дальше узлы общаются
напрямую (а при «злом» NAT — через relay, но трафик всё равно зашифрован E2E).

### Подключиться к существующей глобальной сети

Укажите адрес одного или нескольких публичных узлов (их `IP:базовый_порт`):

```bash
cargo run -p void-ui -- Имя 7700 --bootstrap=node1.example.com:7700,203.0.113.5:7700
```

Клиент зарегистрируется на этих узлах и заберёт у них список участников. Если
один узел недоступен — пробуется следующий.

### Стать публичным узлом (точкой входа)

Любой, у кого есть «белый» IP или проброшенный порт, может держать точку входа:

```bash
cargo run -p void-ui -- Имя 7700 --public
```

В режиме `--public` приложение:

- поднимает **bootstrap-сервер** (порт `база+5`, т.е. 7705) — помогает новым узлам войти;
- поднимает **relay-сервер** (порт `база+6`, т.е. 7706) — ретранслятор для узлов за симметричным NAT;
- пытается **автоматически пробросить порты** через UPnP/NAT-PMP;
- определяет ваш **внешний IP** через публичные STUN-серверы;
- получает **бонус репутации** за помощь сети (см. [Система репутации](Obsidian/Система%20репутации.md)).

Раздайте друзьям ваш внешний адрес `ВНЕШНИЙ_IP:7700` — его приложение печатает
в лог при старте. На странице **Профиль** значок покажет, что вы — Bootstrap-узел,
а строка статуса предупредит, если входящие порты всё-таки закрыты.

> Чем больше людей держат публичный режим — тем надёжнее и «неубиваемее» сеть.

### Прямые соединения между сетями (NAT-traversal)

Чтобы узлы из разных сетей соединялись **напрямую**, а не через relay:

1. Каждый узел при старте пытается пробросить свои порты через **UPnP** (если
   роутер поддерживает) — не только публичные узлы.
2. **STUN** определяет внешний IP для публичных узлов.
3. Bootstrap-узел **публикует ваш внешний адрес** остальным (видит его как
   источник вашего соединения) — так другие подключаются к вам по внешнему
   `ip:порт`, а не по бесполезному LAN-адресу.
4. Если прямое соединение не вышло (симметричный NAT без UPnP) — трафик идёт
   через **relay** bootstrap-узла (всегда зашифрован E2E).

Для гарантированно прямой доступности — включите публичный режим или вручную
пробросьте порты `7700, 7702–7704` на роутере. Статус «порты открыты/закрыты»
виден на странице **Профиль**.

## Порты и файрвол

Базовый порт по умолчанию — **7700** (меняется вторым аргументом). От него
отсчитываются остальные:

| Порт | Протокол | Назначение | Нужен для |
|---|---|---|---|
| `база`   (7700) | TCP | Сервер чанков (файлы/сайты) | хранилище, сайты |
| `база+1` (7701) | UDP | Обнаружение (broadcast) | автопоиск узлов |
| `база+2` (7702) | TCP | Общий чат | чат |
| `база+3` (7703) | TCP | Личные сообщения (DM) | личный чат |
| `база+4` (7704) | TCP | HTTP-сервер сайтов | сайты |
| `база+5` (7705) | TCP | Bootstrap-сервер | только режим `--public` |
| `база+6` (7706) | TCP | Relay-сервер | только режим `--public` |
| 5353 | UDP | mDNS | автопоиск в общей LAN |

Серверы слушают на всех интерфейсах (`0.0.0.0`), поэтому подключение по
VPN-адресу (Hamachi/Radmin) работает так же, как по обычному LAN-IP.

**Минимум для чата:** открыть `UDP 7701` и `TCP 7702`.
**Для файлов/сайтов/ЛС:** добавьте `TCP 7700, 7703, 7704`.

### Windows

При первом запуске Windows Defender покажет окно — нажмите **«Разрешить
доступ»** для частных *и* публичных сетей. Если окно пропустили: *Параметры →
Сеть → Брандмауэр Защитника Windows → Разрешить работу с приложением* и добавьте
`void-connect.exe`.

### Linux

Подсказки прямо в приложении: **Профиль → «Инструкции по настройке файрвола»**.
Кратко:

```bash
# Ubuntu / Debian
sudo ufw allow 7701/udp
sudo ufw allow 7700,7702,7703,7704/tcp

# NixOS (/etc/nixos/... networking)
networking.firewall.allowedUDPPorts = [ 7701 ];
networking.firewall.allowedTCPPorts = [ 7700 7702 7703 7704 ];
# для публичного узла добавьте TCP 7705 7706
```

## Возможности подробно

- **Общий чат.** Сообщения рассылаются по сети и подписываются вашим ключом —
  подделать автора нельзя. История хранится локально в SQLite и подгружается при
  старте. Есть **каналы** («доски»): глобальный + тематические (Технологии,
  Разработка, Игры, Музыка, Кино, Творчество, Помощь, Random, Барахолка).
  Канал — это тег сообщения: всё рассылается всем и фильтруется в интерфейсе,
  а сам тег входит в подпись (нельзя переложить сообщение в чужой канал).
- **Личные сообщения.** Прямое P2P-соединение со сквозным шифрованием (X25519 +
  ChaCha20-Poly1305). Если прямое соединение невозможно — доставка через relay,
  но содержимое остаётся зашифрованным.
- **Хранилище.** Файл бьётся на чанки, каждый проверяется по SHA-256. Скачивание
  идёт параллельно с нескольких узлов; скачавший автоматически становится новым
  сидером. Видны прогресс и число раздающих.
- **Сайты.** Опубликуйте папку со статическим сайтом (HTML/CSS/JS) — он раздаётся
  по той же чанковой схеме и доступен по адресу `имя.void`. Чужие сайты кэшируются
  автоматически при открытии, а кнопкой **«Кэшировать»** можно зеркалировать сайт
  целиком (скачать все файлы и стать его сидером) — тогда он остаётся доступен,
  даже когда автор офлайн. Выбор зеркал переживает перезапуск.
- **Внутренний DNS (`.void`).** Имена подписываются ключом владельца; имя
  достаётся тому, кто заявил его первым. Записи синхронизируются между узлами.
- **Репутация.** Учитывает аптайм, отдачу файлов, валидность чанков, хостинг и
  держание публичного узла; штрафует за спам/флуд, битые чанки и жалобы. Низкая
  репутация — жёстче лимиты, отрицательная — изоляция.
- **Безопасность.** Идентификация по криптоключам (защита от Sybil), rate-limiting
  от флуда, подписи сообщений и DNS-записей, проверка чанков по хэшу.
- **Проверка контактов (safety number).** Имена в сети не удостоверены — кто угодно
  может назваться так же. Настоящая личность собеседника — его ключ. В профиле узла
  есть раздел «Проверка контакта» с **40-значным кодом** (BLAKE3 от пары ключей, как
  у Signal): сверьте цифры с собеседником по другому каналу (звонок, встреча) и
  отметьте контакт проверенным. Если код позже изменится — возможна подмена личности.
  Список проверенных хранится только локально и в сеть не уходит.
- **Опциональный пароль на аккаунт.** Ключи на диске можно зашифровать паролем
  (Настройки → «Безопасность аккаунта»): KEK = KDF(machine-id + пароль),
  PBKDF2-HMAC-SHA256. Без пароля — авто-разблокировка как раньше; с паролем он
  спрашивается при запуске, и изъятия компьютера недостаточно для доступа.
  **Важно:** включение/снятие пароля не меняет ваш ID; но забытый пароль означает
  потерю аккаунта на этом устройстве (восстановить нечем).

## Флаги командной строки

```
void-connect [ИМЯ] [ПОРТ] [ФЛАГИ]
```

| Аргумент / флаг | Значение |
|---|---|
| `ИМЯ` (1-й позиционный) | Отображаемое имя узла (до 32 символов). Перекрывает сохранённое. |
| `ПОРТ` (2-й позиционный) | Базовый порт, по умолчанию `7700`. |
| `--local`, `-l` | Локальный режим (`127.0.0.1`) — для запуска нескольких копий на одной машине. |
| `--public` | Публичный режим: bootstrap + relay-сервер, UPnP, STUN, бонус репутации. |
| `--bootstrap=host:port[,host:port]` | Адреса точек входа для подключения к глобальной сети. |
| `--data-dir=ПУТЬ` | Папка данных (ключи, профиль, БД). То же — через `VOID_DATA_DIR`. |

Примеры:

```bash
# Обычный запуск в LAN/VPN
cargo run -p void-ui -- Vasya

# Две копии на одном ПК (для тестов) — разные порты и папки данных
cargo run -p void-ui -- Alice 7700 --local --data-dir=/tmp/alice
cargo run -p void-ui -- Bob   7710 --local --data-dir=/tmp/bob

# Публичная точка входа
cargo run -p void-ui -- Hub 7700 --public

# Подключение к глобальной сети
cargo run -p void-ui -- Vasya 7700 --bootstrap=203.0.113.5:7700
```

## Где хранятся данные и привязка к устройству

По умолчанию всё лежит в `~/.config/void-connect` (можно сменить флагом
`--data-dir` или переменной `VOID_DATA_DIR`):

- `profile.json` — имя, аватар, статус, описание;
- `keystore.json` — зашифрованный master-секрет (ваш постоянный ID = публичный ключ);
- `void.db` — история чата, индекс файлов, репутация;
- `chunks/` — локальные чанки; `downloads/` — скачанные файлы.

> 🔐 **Привязка к устройству.** Аккаунт привязывается к машине (по machine-id).
> Если просто скопировать папку аккаунта на другой компьютер — доступ
> заблокируется, и приложение покажет экран с пояснением. Удалите чужие файлы
> аккаунта и перезапустите — будет создан новый аккаунт.

> 🔑 **Пароль (опционально).** По умолчанию ключи расшифровываются автоматически
> на этом устройстве. В *Настройки → «Безопасность аккаунта»* можно задать пароль:
> тогда `keystore.json` шифруется ключом, выведенным из machine-id **и** пароля, и
> пароль спрашивается при каждом запуске. Включение/снятие пароля не меняет ваш ID.
> Забытый пароль восстановить нельзя — аккаунт на этом устройстве будет потерян.

## Структура проекта

Cargo workspace из нескольких крейтов:

```
crates/
  core/        Общие типы: NodeId, профили, сообщения, DNS-записи
  crypto/      Ключи, подписи (Ed25519), E2E-шифрование (X25519 + ChaCha20)
  db/          SQLite (sqlx): история, пиры, файлы, репутация
  discovery/   mDNS + UDP-broadcast, bootstrap, NAT (UPnP/STUN), relay
  network/     TCP-транспорт, маршрутизация, rate-limit
  chat/        Общий чат (flood/relay) и личный чат (P2P + E2E)
  storage/     Чанки, SHA-256, скачивание, мульти-сидинг
  web/         HTTP-сервер сайтов, реестр сайтов, внутренний DNS (.void)
  reputation/  Подсчёт очков, события, жалобы, синхронизация
app/           Headless-ядро / точка входа (опционально)
ui/            Графический интерфейс (egui/eframe) → бинарник void-connect
```

Подробное описание модулей — в [Obsidian/Структура проекта.md](Obsidian/Структура%20проекта.md).
